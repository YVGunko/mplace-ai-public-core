# CONTRIBUTING

Спасибо, что хотите помочь развитию **mplace-ai-public-core**.

Этот репозиторий — публичное ядро для AI‑обработки товарных данных маркетплейсов:
типизация, планировщик улучшений по рейтингу и демонстрационный job‑engine.
В нём **нет** интеграций с реальными маркетплейсами и моделями — только абстракции.

---

## Как начать

1. Форкните репозиторий и клонируйте свой форк.
2. Установите зависимости:

```bash
npm install

Работайте в отдельной ветке:
git checkout -b feature/my-change
Структура
packages/core
Типы (ProductSnapshot, RatingEntry, JobPayload, PlannedImprovements и т.п.)
и чистые функции (buildPlannedImprovementsFromRating, shouldRunRich).

packages/job-engine-demo
Простейший in‑memory job‑engine для демонстрации, не для production.

docs/architecture.md
Обзор архитектуры ядра.

Принципы
Без привязки к платформе
Никаких Ozon/WB/Amazon‑специфичных ID, URL, форматов ответов.
Всё платформо‑специфичное должно жить в приватных адаптерах.

Чистые функции и типы
Код в packages/core не должен зависеть от Node.js/Next.js/Express и т.п.

Минимум зависимостей
Старайтесь не добавлять тяжёлые пакеты без реальной необходимости.

Стиль кода
JavaScript / TypeScript (ES modules).
Понятные имена, без аббревиатур, где это возможно.
Предпочитайте небольшие, чистые функции.
Если добавляете новый публичный тип или функцию в packages/core —
обязательно обновите docs/architecture.md.
Тесты
Пока тестов может быть немного или не быть совсем.
Если добавляете нетривиальную логику, пожалуйста, добавьте простой тест
или хотя бы небольшой usage‑пример в docs/ или examples/.

Пул‑реквесты
Кратко опишите задачу и мотивацию изменений.
Покройте новые публичные API документацией в docs/architecture.md или отдельном файле.
Избегайте breaking‑changes без обсуждения в issue.

Спасибо за вклад!
